WHISPER LARGE V3 FINE-TUNING BUG REPORT #3

### Current Error
ValueError: Can only automatically infer lengths for datasets whose items are dictionaries with an 'input_ids' key.

### Error Chain & Symptoms
1. Data processing and collation now working correctly
2. Model loads successfully
3. Error occurs during baseline evaluation
4. Specifically fails in trainer's eval dataloader creation
5. Issue appears to be with length-based grouping of samples

### Relevant Code Areas

1. Training Arguments Configuration:
```python
training_args = Seq2SeqTrainingArguments(
    # ... other args ...
    group_by_length=True,  # This is causing the issue
    dataloader_num_workers=0,
    # ... other args ...
)
```

2. Dataset Structure (Current):
```python
# Our dataset items look like:
{
    "input_features": tensor(...),  # mel spectrogram features
    "labels": list  # token ids
}

# Trainer expects for grouping:
{
    "input_ids": list,  # token ids for length calculation
    # ... other fields
}
```

3. Data Collator Output:
```python
def __call__(self, features):
    batch = {
        "input_features": torch.stack(...),
        "labels": tensor(...)
    }
    return batch
```
